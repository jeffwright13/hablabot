## Project Overview

HablaBot is a **Progressive Web App (PWA)** for AI-powered Spanish conversation practice. It is a pure client-side application — no build system, no package manager, no framework. All JavaScript is loaded via `<script>` tags in `index.html` and exposes globals on `window`.

## Running the App

The app requires HTTPS (or localhost) for the Web Speech API. Serve with any static file server:

```bash
python3 -m http.server 8000
# or
npx serve .
```

Then open `http://localhost:8000` in Chrome or Edge (best speech recognition support).

There is no build step, no test runner, and no linter configured.

## Architecture

### Script Loading Order (index.html)

Scripts are loaded in dependency order — this is critical:

1. `js/utils/config.js` — sets `window.HablaBotConfig`
2. `js/utils/helpers.js` — sets `window.H` (DOM utility namespace)
3. `js/utils/user-manager.js` — sets `window.HablaBotUserManager`
4. `js/storage/database.js` — sets `window.HablaBotDB`
5. `js/ui/state.js` — sets `window.HablaBotUIState`
6. `js/ui/components.js` — sets `window.HablaBotComponents`
7. `js/speech/recognition.js` — sets `window.HablaBotSpeechRecognition`
8. `js/speech/synthesis.js` — sets `window.HablaBotSpeechSynthesis`
9. `js/ai/prompts.js` — sets `window.HablaBotPrompts`
10. `js/ai/conversation.js` — sets `window.HablaBotConversation`
11. `js/vocabulary/spaced-repetition.js` — sets `window.HablaBotSpacedRepetition`
12. `js/vocabulary/manager.js` — sets `window.HablaBotVocabulary`
13. `js/utils/audio.js` — sets `window.HablaBotAudio`
14. `js/app.js` — sets `window.HablaBotApp` (main controller, initialized on DOMContentLoaded)

### Key Patterns

- **No modules**: All files use the global `window` namespace. Each file creates one global singleton.
- **`window.H`**: The helper/DOM utility object used throughout (e.g., `H.$()`, `H.show()`, `H.hide()`, `H.showToast()`, `H.escapeHtml()`).
- **Multi-user support**: Each user gets their own IndexedDB database. `HablaBotUserManager` manages user profiles in `localStorage`, and `HablaBotDB.init()` opens the correct per-user database at login time.
- **Config storage**: Settings saved to `localStorage` under key `hablabot-config`. API key is stored there (not encrypted despite the README claiming otherwise).

### Data Flow for a Conversation Turn

1. User holds push-to-talk button → `HablaBotApp.startListening()` → `HablaBotSpeechRecognition.start()`
2. Speech result fires `onResult` callback → `HablaBotApp.handleSpeechResult()`
3. Transcript sent to `HablaBotConversation.processUserInput()` which calls OpenAI GPT-4 directly from the browser
4. AI response returned → `HablaBotApp.addMessageToHistory()` + `HablaBotSpeechSynthesis.speak()`
5. Synthesis `onEnd` fires → `HablaBotApp.beginUserTurn()` (auto-triggers next listen cycle)

### Storage

- **IndexedDB** (`HablaBotDB`): Vocabulary items, conversation sessions, conversation history. Per-user databases named `{userId_prefix}HablaBotDB`.
- **localStorage**: User profiles (`hablabot_users`), current user ID (`hablabot_current_user`), app config (`hablabot-config`).

### AI Integration

`js/ai/conversation.js` calls `https://api.openai.com/v1/chat/completions` directly from the browser using the user's OpenAI API key. The model is hard-coded as `gpt-4`. The system prompt is generated by `js/ai/prompts.js`.

### Speech

- **Recognition**: Uses browser-native `window.SpeechRecognition` / `window.webkitSpeechRecognition`. Language defaults to `es-ES`. Has retry logic for network errors.
- **Synthesis**: Uses browser-native `window.speechSynthesis`. ElevenLabs is a planned fallback (feature-flagged off via `config.enableElevenLabs`).

### PWA

`sw.js` is the service worker, registered in `index.html`. `manifest.json` defines PWA metadata. Icons are missing (only a `placeholder.txt` in `assets/icons/`).
